# Context 哲学与 commit 工作流 待确认问题

## 高优先级

- 是否要继续设计一个与 `/commit` 配套的 `/load-context` 工作流
- `/commit` 后续是否需要进一步做成更自动化的执行脚本或命令
- 用户希望系统在什么情况下主动提醒“这次可能值得 commit”

## 中优先级

- 是否要把“context 哲学”进一步扩展为团队级或公司级工作流
- 是否要把私密情绪材料与工作 context 做更严格的权限分层
- 是否要为不同 agent 定义一套更统一的加载入口

## 已确认

- 用户高度重视 context，不愿意只比较模型而忽略 context
- 用户希望 Git 作为跨 agent、跨设备、跨模型的 context 底座
- 用户认为 `/commit` 应该按长期增量触发，而不是按日期机械执行
