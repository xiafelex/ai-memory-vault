# 表达风格与修改标准归纳 证据与原话

## 原话摘录

- 原话：每个素材部分增加 AE 左上角标题，类似原来投标阶段视频。
  解释：用户非常重视结构化提示和层次感，不希望内容裸奔。
  分类：结构要求 / 视频表达

- 原话：不要放太多例如 cursor 页面，钉钉页面，表格页面的素材，让人看不明白到底要表达什么。
  解释：用户明确反对信息密度低、理解效率差的“页面堆砌型”表达。
  分类：素材有效性 / 观众理解

- 原话：每一页的标题，就是和大章节相同的标题放到左下角和 logo 同样高度的位置，把二级标题放在一级标题中。
  解释：用户对页面结构、视觉秩序和层级关系有明确要求。
  分类：PPT 结构 / 排版标准

- 原话：按照问题来分章节，不要按照人来分章节。
  解释：用户表达时更重任务和主题组织，而不是按人物或素材来源堆叠。
  分类：结构原则 / 采访内容组织

- 原话：每一页要有小标题和图片，还有文字简述。
  解释：用户希望单页表达具备完整的信息单位和快速理解能力。
  分类：PPT 标准 / 页面任务

- 原话：每秒获取到的信息，应该是有效而独立的，不要重复。
  解释：这是用户对视频信息密度最关键的判断之一。
  分类：节奏标准 / 素材筛选

- 原话：应用重难点展示，要跟随字幕进行高斯影像的对应素材选择，例如“布局拥挤”，画面就要体现布局拥挤。
  解释：用户对“画面必须精确对应文案含义”的要求非常高。
  分类：画面与文案一致性 / 表达精确度

- 原话：PPT 中 BIM 实施目标与计划与实际工作不符合，这个需要贴合实际情况来写。
  解释：用户反对脱离实际的表述，强调内容真实贴合工作。
  分类：真实性 / 任务适配

- 原话：PPT 排版还是过于简单，要多有层次的章节。
  解释：用户不接受平铺直叙、缺乏层次感的表达。
  分类：结构感 / 排版要求

- 原话：不要放进度模拟，涉及到正版版权问题。
  解释：用户在表达中也会考虑合规、版权和边界问题，而不是只看效果。
  分类：合规意识 / 内容边界

## 可直接作为长期证据的内容

- 用户长期稳定强调“结构、层次、章节、标题”
- 用户长期稳定强调“素材有效性”和“理解效率”
- 用户长期稳定强调“画面、字幕、数据、配音一致”
- 用户长期稳定强调“真实性、代表性、成果感”

## 需要后续核实的点

- 用户在纯文字型材料中，是否比视频/PPT允许更高密度的信息组织
- 用户是否更偏好“克制冷静型”还是“强节奏高冲击型”的对外视频风格
