# 构建 AI 记忆 Git 框架 摘要

## 这次对话解决了什么

- 从零创建了一个基于 Git 的 AI 记忆仓库
- 明确了仓库的目标：把重要对话保存下来，并让后续模型复用这些上下文
- 完成了本地仓库初始化、基础 CLI、长期画像模板、上下文导出流程
- 完成了 GitHub 仓库连接与首次推送

## AI 后续应该记住什么

- 用户当前最关心的是建立跨模型、跨平台可迁移的 AI 记忆层
- 用户希望把重要对话沉淀为长期资产，而不是停留在单次聊天窗口
- 用户对 GitHub 的使用经验还在建立中，后续引导应尽量具体
- 用户希望 AI 主动帮助总结、分类、结构化整理，而不是只给方向

## 可复用约束 / 偏好 / 决策

- 优先选择“先可用、再增强”的实现路线
- 优先使用 Git 仓库作为用户拥有的数据底座
- 默认用中文交流
- 后续所有模型都应尽量复用 `memory/profiles` 和 `memory/snapshots/active_context.md`
