# 跨电脑 / 新对话继续同一主题的工作流

## 适用场景

当用户在不同电脑、不同时间、不同对话中继续同一个主题时，应尽量避免重新解释背景和重复整理已有结论。

典型场景包括：

- 继续某个项目策划
- 继续某个项目文稿优化
- 继续某个课题材料整理
- 继续某个长期协作主题

## 默认继续方式

如果用户在新对话中说：

- 继续开始马来西亚和德西尼布关于 SAF 新项目的策划
- 继续优化上次的项目策划
- 继续昨天那个主题

则默认不应从零开始，而应先做以下判断：

1. 用户是否给出了主 memory 仓库地址
2. 用户是否指出了要继续的主题名称
3. 用户本轮是否带来了新增资料或新增要求

## 新对话开始时最少需要的信息

后续继续同一主题时，用户最少只需提供：

- 主 memory 仓库地址
- 当前继续的主题名称

如果本轮有新材料，再补充：

- 新上传的文件
- 新的问题或新目标

## AI 的默认动作

当用户提供仓库地址并说明继续某个主题时，AI 应默认做以下几件事：

1. 优先读取该主题对应的 conversation 和 domain 主档案
2. 优先读取最近一次该主题的 summary / open_questions / domain_updates
3. 识别本轮新增内容与旧内容的关系：
   - 是补资料
   - 是改写文稿
   - 是新增分工判断
   - 是从中文方案转英文方案
4. 在已有框架上继续，而不是重写整套背景

## 推荐表达模板

用户后续可以直接这样开启新对话：

- 这是我的主记忆仓库：<git 地址>
- 我们继续马来西亚和德西尼布关于 SAF 新项目的策划
- 本轮我补充了这些资料：...
- 这次先优化哪一部分：...

## 对类似主题的通用规则

后续其他话题也可采用同一模式：

- 主题先在 memory 仓库中形成 conversation 和 domain
- 每一轮只补本轮新增资料和新增目标
- 已有稳定结论不重复展开
- 继续围绕“上轮结论 + 本轮新增”推进

## 目标

这套工作流的目标是：

- 减少重复解释成本
- 保持跨电脑、跨对话的一致性
- 让长期主题具备连续推进能力
- 让主 memory 仓库真正承担主题衔接中心的作用
