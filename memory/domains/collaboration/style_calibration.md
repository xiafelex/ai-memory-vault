# 风格校准场景

这份主档案用于管理“通过持续投喂材料，逐步校准用户个人风格、判断标准和 AI 仿写边界”的固定场景。

## 场景定义

- 这不是一次性下结论的任务，而是一个持续采样、持续修正的过程
- 目标不是只总结“写得好不好”，而是区分：
  - 哪些是文体或场景自带的结构
  - 哪些是用户稳定的个人语言风格
  - 哪些是用户的修改标准
  - 哪些是用户对 AI 润色的纠偏

## 默认流程

1. 读取用户提供的新材料
2. 先判断材料类型和适用范围
3. 明确这次材料主要能支撑哪一层结论
4. 把稳定结论更新到对应主题主档案
5. 把关键原文、典型修改痕迹、待确认点写入当前会话的 `evidence.md` 和 `open_questions.md`

## 常见子场景

- `paper-style`
- `policy-style`
- `bid-video-style`
- `ppt-style`
- `ai-revision-comparison`
- `review-comment-style`

## 典型校准信号

- 用户说“看上去写得很对，其实没有重点”，通常说明内容空泛正确但没有抓手
- 用户指出“懂行的专家一看就觉得不对”，通常说明材料对行业产品能力边界的描述失真
- 用户主动把“高度依赖”改成“依赖”这类表达，通常说明用户对判断强度和形容词分寸有明确要求
- 用户重写不是为了推翻结构本身，而是为了把重点、真实性和分寸重新拉回到自己认可的水平
- 用户认可“折中版本”，通常说明其更偏好“先生成可用稿，再做局部修正”，而不是一次性追求最满的终稿
- 在成果材料场景下，用户更接受“差不多能用，只改局部”的稿子，而不是语言漂亮但需要整体重写的稿子

## 默认输出

- `summary.md`
  这批材料新增了什么较稳定的风格判断
- `evidence.md`
  关键原文、典型句子、典型改法、能证明判断的例子
- `open_questions.md`
  哪些点还无法确认，需要继续补材料
- `domain_updates.md`
  是否更新 `writing_style.md`、`traceable_response_style.md` 等主题主档案

## 判断原则

- 不把“文体要求”误判成“个人偏好”
- 不把单次材料中的偶然措辞放大成稳定风格
- 如果用户明确纠偏，优先以用户的纠偏为准
- 当证据不足时，宁可保守，也不要过度总结
