# 表达风格与写作标准

## 风格原则

- 先明确表达目标，再组织内容
- 结构必须立住，章节、层次、序号和主次关系要清晰
- 观众或读者必须能快速看懂重点，而不是靠猜
- 配音、标题、字幕、画面、术语、数据必须彼此一致
- 素材必须有代表性和有效信息密度，避免无效画面
- 节奏可以快，但关键内容必须让人看清、看懂
- 优先呈现成果、亮点、数据和具体系统，而不是泛泛而谈
- 有技术和业务含量，能支撑说服力
- 务实，不堆字数，不追求表面华丽

## 常见判断标准

- 这段内容到底在表达什么
- 这一页或这一段是否完成了它应该承担的任务
- 观众能否一眼看出重点
- 当前素材是否比替代素材更能说明问题
- 这部分内容是否与配音或标题真正对得上
- 信息是否太快、太杂、太重复，以至于降低理解效率

## 适用场景下的表现

### 视频 / 配音稿

- 更强调镜头、节奏、素材匹配、章节切换和观众理解
- 不接受无效画面、重复素材、手动录屏抖动、配音与画面脱节
- 要求素材真正对应字幕内容，而不是“差不多有点像”

### PPT / 汇报材料

- 更强调章节感、层次感、排版完成度和页面任务
- 每一页都应有清晰的小标题、图片和简述
- 页面不能过满，也不能只是简单堆砌截图和文字

### 技术竞赛 / 成果申报

- 更强调技术名称、算法点、成果映射、数据支撑和应用逻辑
- 技术亮点必须被明确标出来，而不是埋在叙述里
- 重点成果应与章节和标题形成清晰映射

### 管理 / 经营材料

- 更强调框架收敛、章节整合、逻辑秩序和重点聚焦
- 不喜欢散乱平铺，偏向按主题归并和分层表达

## 避免事项

- 套话
- 空话
- 评审腔
- 只为写材料而写材料
- 看起来很多、实则很水的内容
- 无效画面
- 重复素材
- 配音、画面、字幕、数据不一致
- 页面很多但看不出到底要表达什么
- 为了凑时长或凑页数而堆内容

## AI 写作与修改时的默认要求

- 先给结构，再写正文
- 先判断受众和目的，再决定写法
- 必须说明重点，不要把真正重要的东西埋掉
- 如果要给素材建议，要说明“为什么放这个，不放那个”
- 如果要改稿，优先改表达任务、结构和理解效率，再改措辞
