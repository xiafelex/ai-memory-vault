# 项目型输出双轨规则

## 主题定义

这个主档案用于沉淀项目型任务的输出规则，明确区分“思路整理版”和“正式交付版”两类文件，避免分析过程与汇报文本混写。

这不是一次性的写作偏好，而是后续项目策划、管理方案、汇报材料、对外交流稿都可重复使用的稳定规则。

## 稳定规则

### 1. 思路整理版

思路整理版用于保存分析过程、判断依据、证据来源、当前建议和待补资料。

它的特点是：

- 可以保留推导过程
- 可以写“当前判断”“建议”“待确认项”
- 可以记录证据来源、原始文本归纳、差异分析
- 适合持续迭代、补资料和内部讨论

适用场景：

- 项目前期研究
- 方案比较
- 证据沉淀
- 内部分析
- 继续加工前的工作底稿

建议命名：

- `*_working.md`
- `*_thinking.md`
- `*_planning.md`

### 2. 正式交付版

正式交付版用于直接汇报、群发、对外交流或提交领导审阅。

它的特点是：

- 不保留对话痕迹
- 不写“它解决的是这件事”“当前定位”“建议先这样做”这类引导性表述
- 直接按正式文件结构写成完整文稿
- 更强调背景、目标、安排、分工、措施和结论

适用场景：

- 汇报材料
- 群发版文件
- 领导审阅稿
- 对外交流稿
- 可直接提交的正式文本

建议命名：

- `*_delivery.md`
- `*_report.md`
- `*_formal.md`

## 两类文件的关系

同一主题下，原则上允许同时存在两类文件：

- 一份思路整理版，负责保留判断过程和可迭代内容
- 一份正式交付版，负责形成可直接使用的正式文本

二者不是互相替代关系，而是上下游关系：

- 思路整理版负责长内容和分析深度
- 正式交付版负责最终表达和交付适配

## 使用建议

对于需要持续推进的项目型主题，推荐采用以下工作流：

1. 先形成思路整理版，沉淀事实、证据、判断和待补资料。
2. 在思路稳定后，再单独形成正式交付版。
3. 如果后续补充了新资料，优先更新思路整理版。
4. 当汇报口径变化或需正式发送时，再同步更新正式交付版。

## 当前适用示例

“马来西亚生物柴油项目与 Technip 分工策划”主题已经适用该规则：

- `malaysia_biodiesel_project_planning.md` 和相关主档案可视为思路整理版体系
- `malaysia_biodiesel_project_planning_report.md` 可视为正式交付版

## 后续默认执行方式

后续涉及项目策划、管理方案、工作计划、技术路线、汇报材料等任务时，默认按双轨规则输出：

- 先判断是否需要长期沉淀分析过程
- 再判断是否需要直接形成可交付文本
- 若两者都需要，则同步保留两份文件
