## 主题结构

### 1. 管理组织与制度

包括管理创新组织、技术中心部门及岗位职责、管理制度交流会、项目不同阶段管理方法及部门责任划分、研发项目验收及付款流程、技术服务合同认定税务问题等。

这一组内容适合作为技术中心管理体系的基础层，后续应沉淀成：

- 部门职责和岗位职责索引
- 项目阶段管理责任划分
- 研发项目验收与付款流程
- 管理制度会议材料和待办清单

### 2. 组织建设

包括组织变革、组织问卷、组织现状、数据科学中心化团队建议、国内统一管理、海外项目驻国内期间管理办法、月度管理职能工作汇报等。

这一组内容适合沉淀成组织管理和团队建设主线，重点关注：

- 技术中心组织变革过程
- 管理职能工作月度复盘
- 国内外组织协同与驻国内管理
- 数据科学团队或中心化能力建设

### 3. 研发管理办法

这是目前文件最多的主题，包含研发管理办法、研发资金使用、研发费用加计扣除、技术中心/工程技术研究中心、科技型中小企业、高新技术企业、课题层级与架构、科技论文和成果申报等。

这一组应作为技术中心研发管理制度库的核心，后续适合拆成：

- 研发管理制度
- 研发经费和税务政策
- 课题申报和验收
- 企业技术中心与工程研究中心
- 科技成果、论文和人才激励政策

### 4. 成果竞赛与科技成果材料

包括 AI+应用成果、南京市重大科技专项、安装领域科技成果评价、安全外审、BIM 管道预制排产技术报告、IDF 文件架构解析、应用证明、智能建造技术导则等。

这一组内容适合服务成果申报、奖项材料、技术报告复用和证明材料沉淀。
后续应优先把成果名称、申报类型、证明材料、应用项目和技术亮点做成结构化索引。

### 5. 数字化代码和管理工具

包括数据组日志、焊接日报归集、氚云表单、ERP 视图、OA 加班申请、薪资管理、研发课题申报等代码或流程记录。
管理工具中还包括氚云模块标准化、配对对比法、要素记分法、工程审计激励理论等。

这一组内容适合沉淀为部门数字化管理工具库，后续建议按照“业务场景、数据来源、脚本/表单、输出结果、维护人”建立索引。

### 6. 产业工人与人力资源

包括岗位需求及培养计划、职业技能竞赛、招聘海报、分公司负责人、人力资源年审、MAXHUB 操作教程等。

这一组内容适合作为技能人才、培训和人力资源协同资料库。

### 7. 成本价值与标准规范

成本价值包括工程结算、进度款审批、焊接日报生成产值报表、氚云表单数据读写等。
标准规范包括 BIM 技术标准、石油库设计规范、石油化工企业设计防火标准、核心期刊检索等。

这一组内容适合服务管理价值核算、工程结算、标准查询和资料检索。

## 同步路线判断

本知识库后续不适合只依赖钉钉开放 API。
推荐路线是：

1. API 抓目录、元数据、普通段落、标题和表格。
2. API 结果中保留缺失块标记和原始 alidoc 链接。
3. 对缺失列表项、代码块、图片、同步块和附件多的文档，使用 PDF/浏览器渲染兜底。
4. 在 Git 中分层保存：
   - `memory/imports/` 保存清单、原始块审计、导入文本和证据；
   - `memory/domains/management/` 保存稳定主题主档案；
   - 原始大文件或完整导出件视体积和复用价值决定是否进入 `Felex 资料/`。
