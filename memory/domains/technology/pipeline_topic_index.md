# 石化管道数字化课题索引

这份索引用于给“石化管道数字化管控与智能决策关键技术研究与应用”建立一个稳定入口，避免后续在大课题综述、阶段成果正文、项目过程资料、成果评价附件和聊天沉淀之间来回翻找。

## 1. 这份索引解决什么问题

当前这条主线不是单一文档，而是几套并行材料共同支撑：

- 大课题总论：说明这条长期研发主线为什么成立、边界是什么、阶段如何演进。
- 阶段成果正文：说明当前最成熟落点“石化管道预制焊接排产系统”已经做到什么程度。
- 项目过程资料：说明课题如何立项、如何实施、如何中期与结题管理。
- 成果评价资料：说明面向江苏省安装行业协会科技成果评价时，需要提交哪些正式材料。
- 会议纪要与聊天沉淀：说明很多关键判断并不是事后包装，而是从持续研发与项目服务中逐步形成的。

后续凡是继续写申请书、成果报告、汇报材料或补支撑件，优先从这份索引进入。

## 2. 推荐阅读顺序

1. 先看大课题总论，明确上位叙事。
2. 再看当前阶段成果正文，明确申报书核心章节怎么写。
3. 再看过程管理与成果评价逻辑，明确材料该归到哪一层。
4. 最后补会议纪要、正式通知和聊天沉淀，作为支撑与追溯依据。

## 6. 一条清晰的逻辑链

建议以后始终按下面这条链理解这批材料：

大课题主线
-> 当前成熟切口“石化管道预制焊接排产系统”
-> 七章节阶段成果正文
-> 中施企协项目过程资料与苏安协成果评价资料双轨适配
-> 会议纪要、项目案例、知识产权、用户意见等支撑材料

换句话说：

- 大课题总论负责讲“这件事是什么、为什么值得长期做”。
- 七章节正文负责讲“当前这一步已经做成了什么”。
- 双轨资料逻辑负责讲“这套内容分别怎么投到不同协会和不同阶段”。
- 会议纪要和案例负责讲“这些内容不是凭空写出来的，而是从实践里长出来的”。

## 7. 以后新增材料的归档规则

后续新增文档时，优先判断它属于哪一类：

1. 大课题总论材料
2. 阶段成果正文材料
3. 项目过程管理材料
4. 成果评价附件材料
5. 支撑与证据材料
6. 聊天沉淀与修改记录

如果一份材料同时服务两边，不要重复讲两套不同逻辑，而是：

- 先维护一份“共用技术主线底稿”
- 再根据不同协会或不同申报环节做格式适配
