1. 引言
1.1 项目背景
1.1.1 南京南化建设有限公司承建的某境外生物燃油项目生产装置、罐区、码头和公用工
程的所有地下管线和地上土建、安装、电仪的采购、物流、建设、试车等工作。项
目采用业主、新加坡 PMC、法国国际检验局(Bureau Veritas)与南化建四方共管模
式， 执行美国ASME 与英国 BS 标准。本项目压力管道总长度 10 万米，总焊接量
42万寸。动力、油品储运、锅炉等关键部位受境外职业安全与健康部认证监管，焊
接较中国标准更为严苛，由于存在严格的劳工配额问题，无法大量派遣有高水平压
力管道焊接水平的中国焊工准入，因此项目采用国内预制、国外安装的管理方式，
项目团队针对石化管道 BIM 模型及数据特点，设计了多种管道预制排程新方法确保
预制工作顺利实施。
1.2 BIM 应用目标
1.2.1 管道焊接排产制替代劳务分包制进行预制部署。
1.2.2用最优周期和成本实现管道预制焊接交付，实现材料“先干先到、后干后到”的管理目
标。
1.3 BIM 应用重难点
1.3.1 管道模型载体与数据统一问题。与民用建筑 BIM 适用面广、开放程度高的 Revit 与
IFC 数据标准不同，石化管道设计以 AVEVA、Intergraph、Bentley 等国际流程工业
软件的 rvm、dri、vue 等格式为主，软件部署成本高昂、操作难度大、封闭性强，限
制了石化行业在深化设计和BIM应用领域的软件部署和二次开发，数字化施工的普及
和应用发展较为缓慢。同时石化管道以焊接和达因(Dia-Inch)作为贯穿施工过程的主
要数据，如何打通管道模型与焊接达因的数据关系，并针对性在施工环节进行应
用，成为项目技术准备的重点。
1.3.2 焊缝模型批量定义与优化问题。项目由于生产工艺特点，设计压力管线 2657 条、介

3. 模型数据应用
3.1 模型精细度与完整性情况
3.1.1土建专业使用 Revit 进行建模，主要包括储罐基础、装置框架、地坪管支架定位，生
产辅助用房等建筑单体不在本次 BIM 应用重点，模型深度为 LOD200。
3.1.2 钢结构专业由钢结构供应厂家提供 TeklaStructure 深化模型进行整合，模型深度
LOD300。
3.1.3 设备专业由 Ep3D 与上游设计单位提供的 PDMS 构建，设备专业重点在于坐标与设
备管嘴的准确定位，精度为 LOD200
3.1.4 由于石化管道特殊性，我方主要围绕管道专业延伸上下游进行专业模型构建，重点
在管道本身、其中消防、地管、给排水专业由于不在工艺管道监管范围，并未进行
详细模型构建。
3.1.5工艺管道专业通过Plant3D、Ep3d正向三维配管与idf/pcf逆向建模相结合形式，建立

4. BIM技术基础应用
4.1 构建石化管道焊接多元异构三维模型主数据集
4.1.1 研究并解析 idf 文件中管道元件信息、材料描述、坐标尺寸等元数据架构，建立文本
解析算法（Text Parsing Algorithm）、数据抽取算法（Data Extraction
Algorithm）、数据结构化算法（Data Structuring Algorithm）、几何建模算法
（Geometric Modeling Algorithm）、材料属性映射算法（Material Property
Mapping Algorithm）、焊缝生成算法（Weld Generation Algorithm）、逆向建模算
法（Reverse Engineering Algorithm）等多种算法程序，实现逆向建模生成焊缝模
型，解决了Sk0ey 映射失败、未定义材料编码、元件重叠、错位、不连通等 idf 信息
缺失问题，建立BIM、施工、合同多部门基于钉钉事件流的管道变更协同核对机制流
程，同步变更文件在主模型文件中的修改及数据更新，首次实现对于设计模型文件
的自主更新能力。累计改进文件163 份，融合变更管线87条，确保基础数据质量有
效性。
