1.2 标签库构建
标签库的构建是实现材料编码标准化的基础。采用
了正则表达式 （Regex） 技术 ［3］

匹配， 提取出材料的类型、 尺寸、 材质等信息， 并将其映射
到相应的标签。通过不断扩充标签库， 可以覆盖项目中
大部分常见的材料描述。然而， 仅靠正则表达式无法完
全处理所有复杂的材料描述。例如， 某些材料描述中可

表达式过分依赖于空格等间隔符号， 无法准确识别 “碳钢
法兰” 是属于 “碳钢” 和 “法兰” 两种不同属性。因此， 有必
要进一步引入自然语言处理技术， 对材料描述进行更为
精准的解析和编码。

2.2 NER模型训练
基于 BERT 预训练模型构建了命名实体识别
（NER） 系统， 用于识别复杂的材料描述。BERT 模型强
大的语言理解和上下文捕捉能力， 使其在处理长尾数据
和不规则材料描述时具备极高的准确率。模型训练依托
Pytorch和transformers ［4］ 框架， 主要流程如下。
（1） 数据输入与序列化： 在NER任务中， 输入模型的
文本必须经过适当的预处理。首先， 标注好的材料描述
文本通过分词器 （Tokenizer） 进行切分， 生成对应的
token 序列。由于不同的材料描述存在长度差异。例如
“90 度 长 半 径 弯 头 ，GB/T8163-20，BE，SH/T3408，
SMLS， DN200， S-STD” 含有23个token， 而 “带颈对焊法
兰 ，ASTM，A182 Grade F304（UNSS30400），RF，
CL150， ASMEB16.5， DN80， S-10S” 只有19个token， 必
须对序列进行统一处理。为此， 模型在输入之前需要对
序列进行padding操作， 将较短的序列填充至同一长度。
填充部分使用特殊符号 ［PAD］， 以确保批处理操作时能
够在相同维度上计算， 为了确保这些填充符号不会干扰
模型的训练和推理， 使用了注意力掩码 （attention
mask）。该掩码机制会将实际的token标记为1， 填充部
分标记为0。
图4 材料代码Token/Padding生成及共现矩阵
（2） 嵌入层与特征向量化处理： 完成padding和掩码
处理后， 文本数据进入嵌入层 （Embedding Layer）， 其中
每个 token 被转化为高维向量表示 （embedding）。在这
个过程中， BERT 模型结合了上下文和位置信息， 为每个
token生成语义丰富的特征表示。由于材料描述中的字
段结构复杂 （如标准、 材质、 尺寸等混杂）， BERT 的嵌入
层能够有效地将这些词汇映射到高维向量空间， 确保实
体的多维语义信息能够被准确捕捉。BERT 模型通过多
头注意力机制 （multi-head attention） 进一步处理这些
嵌入表示。多头注意力机制允许模型在多个维度上独立
计算token之间的相关性， 从而捕获序列中的长距离依赖
关系。对于材料描述这类信息密集且字段分散的输入，
模型能够有效识别出不同属性之间的复杂关联。例如，
输 入 材 料 描 述“PIPE， ASTM A106 GRADE B，
SCH20 DN600”， 模型不仅需要识别每个词的独立语义，
还需要关注 “ASTM A106” 作为一个整体材料标准的特

3.1 流水号自动编号与唯一编码生成
结合NER模型生成的标签数据， 系统进一步优化了
管道的自动编号流程。管道焊口和元件的自动编号依赖
于管道流向、 主管与支管的关系。这种基于拓扑关系的
自动编号技术通过解析管道的连接和流向， 自动生成流
水号， 实现了从材料描述到管道编号的统一性， 确保了项
目管理中的数据一致性和追溯性。流水号不仅适用于焊
口表的展示， 还可写入模型的属性中， 形成最终的材料唯
一编码。
例如， 针对多个相同的三通元件， 系统会依次生成
“材料代码-01”“材料代码-02” 等编号， 确保每个元件的
唯一性。在直管段分段的情况下， 每个分段都会获得不
同的材料编码和流水号。通过这种方式， 系统确保了元
件和焊口在整条管线中的唯一标识。
