# DingTalk 日志长期存放规则

这层目录用于保存适合长期进入 Git 和记忆系统的钉钉日志原始证据。

## 目录结构

推荐按以下层级存放：

- `memory/imports/dingtalk_logs/<组织口径>/<年份>/<月份>/`

例如：

- `memory/imports/dingtalk_logs/设计研究院_南京/2026/2026-04/`

## 文件命名

推荐统一使用：

- `<姓名>__<userid>.md`

例如：

- `成哲辉__290752666924835815.md`
- `吴俣__013944274106688687.md`

这样做的原因：

- 姓名变动或同名时，`userid` 仍可稳定追溯
- 后续脚本、索引和知识库提取时，可以按 `userid` 唯一定位
- 不必把部门名重复塞进文件名，部门归属应由目录层和索引层承载

## 分层原则

这一层只保存：

- 原始日志 Markdown

不在这里保存：

- 抓取统计说明
- 批次名单
- 组织口径说明

这些应分别放到：

- `memory/imports/日志抓取记录/`
- `memory/imports/组织架构/`

## 与 outputs 的关系

- `outputs/` 可以继续作为临时工作层和批量导出层
- 真正需要长期进入 Git 的日志证据，应逐步整理到本目录
